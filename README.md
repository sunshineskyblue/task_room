# Life in Pocket
宿泊物件の貸し借りサービス(airbnb等)を模倣した模擬サイトです。</br>
自分なりに宿泊サイトが抱える課題を設定し、その解決を試みました。</br>
airbnbをはじめ、多くの宿泊サイトにはレビュー機能が備わっていますが、現状のレビュー機能には、</br>
以下の課題があると感じました。</br>
<br />

- **価格がサービスの品質や満足度に影響を与える因子となり得るにもかかわらず、</br>
あらゆる価格帯の宿泊物件が星の数という同じ物差しで比較されてしまう。**</br>

- **コメント機能はあるが、その他にも感情に訴える強い思い、感動を表現することができない。</br>
つまり、星の数という制約がある。**</br>
<br />

このアプリケーションでは、それぞれの解決策として以下２つのアプローチで解決を試みました。</br>
<br />

- **宿泊価格に応じて、その価格帯における平均スコアを可視化する**
<img width="985" alt="スクリーンショット 2023-01-02 20 19 51" src="https://user-images.githubusercontent.com/109893844/210368860-6aff7300-3abd-4d59-a0f2-4b45caa11eda.png">

 =>  **平均が見える化されることで、ユーザーは星の数という絶対値だけでなく、平均スコアとの比較が可能となり、</br>
     <span>　</span>各価格帯の中で相対的な視点を持つことができる**
<br />
<br />

- **年に一回しか送ることできない、いいね機能**</br>

![ezgif com-gif-maker (1)](https://user-images.githubusercontent.com/109893844/210483516-3745ce5d-f235-4053-a87b-e6c6c18878fc.gif)</br>

 =>  **年に一回という重みをつけることで、星５を超える「何か」があることを視覚的に訴えられる。**
<br />
<br />

# サイトイメージ
Life in Pocketは、宿泊サイトとしての基本機能も備えており、</br>
一つのアカウントでホストとして物件の登録、またゲストとして宿泊予約が可能です。</br>
ゲストログイン機能を備えているため、どなたでもこのサービスをご利用頂けます。</br>
<br />

# 宿泊サイトを選んだ理由
制作期間を2ヶ月と定め、その中で幅広いrailsの機能に触れ、railsの知識、スキルを高めることを第一の目標と致しました。</br>
宿泊サイトでは、予約、通知、レビューと言った連動が求められるCRUD機能を始め、</br>
インターアクティブな操作が伴うことから、それらの実装を通して、railsを深く、広く学びことができると考え、</br>
今回、宿泊サイトをポートフォリオとして選びました。</br>
<br />

# こだわりポイント
- **キャンセル・キャンセルリクエスト機能**</br>
インターアクティブな操作として、ゲスト・ホストの双方から予約に対してキャンセルすることが可能です。</br>
ホストの場合、キャンセルリクエストという形でゲストにリクエストを送り、ゲストからの承認を経てキャンセルが実行されます。</br>
通知機能とも連動し、予約時含めキャンセルあるいはキャンセルリクエストが入った場合は、それぞれに通知が入ります。
